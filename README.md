# Knowledge Indexer (`ki`)

Index knowledge sources (currently: Git repositories) into a 3-layer hierarchy — **Source → Feature → FunctionUnit** — designed for AI-agent consumption. Summaries are generated by a server-side LLM (Anthropic Claude **or** any OpenAI-compatible endpoint, including self-hosted models like Ollama, vLLM, and LM Studio); the result is persisted as JSON and exported to a single `KNOWLEDGE.md` (and optionally YAML).

## Requirements

- Node.js ≥ 20
- One configured LLM provider:
  - `ANTHROPIC_API_KEY` (Claude), **or**
  - `OPENAI_BASE_URL` + `OPENAI_MODEL_ID` (with `OPENAI_API_KEY` for hosted providers)

## Install

```bash
npm install
npm run build
npm link        # makes the `ki` binary available globally
```

For local development, `npm run dev -- <args>` runs the CLI through `tsx` without building.

## Configuration

Configuration is read from environment variables. You must configure exactly one provider.

### Provider selection

| Variable | Default | Description |
| --- | --- | --- |
| `KI_LLM_PROVIDER` | _(inferred)_ | `anthropic` or `openai`. When unset, inferred as `openai` if `OPENAI_BASE_URL` or `OPENAI_API_KEY` is set; otherwise `anthropic`. |

### Anthropic provider

| Variable | Default | Description |
| --- | --- | --- |
| `ANTHROPIC_API_KEY` | _(required)_ | API key for Claude |
| `KI_ANTHROPIC_MODEL_ID` | `claude-sonnet-4-20250514` | Anthropic model id (legacy `KI_MODEL_ID` still honored) |

### OpenAI-compatible provider

Works with **OpenAI**, **Ollama**, **vLLM**, **LM Studio**, **OpenRouter**, **Together**, **Groq**, and any other server that speaks the OpenAI Chat Completions API.

| Variable | Default | Description |
| --- | --- | --- |
| `OPENAI_BASE_URL` | `https://api.openai.com/v1` | Base URL (path may include or omit `/chat/completions`) |
| `OPENAI_MODEL_ID` | `gpt-4o-mini` | Model id (alias: `KI_OPENAI_MODEL_ID`) |
| `OPENAI_API_KEY` | _(empty)_ | Bearer token; omit for self-hosted servers that don't require auth |

#### Self-hosted examples

**Ollama:**
```bash
export OPENAI_BASE_URL=http://localhost:11434/v1
export OPENAI_MODEL_ID=llama3.1
# no OPENAI_API_KEY needed
```

**vLLM / LM Studio / text-generation-webui:**
```bash
export OPENAI_BASE_URL=http://localhost:8000/v1
export OPENAI_MODEL_ID=your-model-name
```

**OpenRouter:**
```bash
export OPENAI_BASE_URL=https://openrouter.ai/api/v1
export OPENAI_MODEL_ID=meta-llama/llama-3.1-70b-instruct
export OPENAI_API_KEY=sk-or-...
```

### Pipeline options

| Variable | Default | Description |
| --- | --- | --- |
| `KI_DATA_DIR` | `.data/indices` | Where index JSON files are stored |
| `KI_MAX_CONCURRENT` | `5` | Concurrent summarization requests |
| `KI_OUTPUT_FORMAT` | `json` | `json`, `yaml`, or `both` for sidecar export on `index` |
| `KI_MARKDOWN_PATH` | `KNOWLEDGE.md` | Default output path for the markdown export |

See `.env.example` for a full reference.

## Commands

### `ki index <target>`

Scan a target, summarize it via the configured LLM, persist the index, and write `KNOWLEDGE.md`.

```bash
ki index ./my-repo
ki index ./my-repo --type git --include "src/**/*.ts" --exclude "**/*.test.ts"
ki index ./my-repo --dry-run        # scan only, no LLM calls
ki index ./my-repo --force          # re-index even if nothing changed
ki index ./my-repo -o docs/REPO.md  # custom output path
```

Diff detection skips re-summarization when the source hasn't changed since the last run; `--force` overrides this.

### `ki list`

List all sources currently indexed in `KI_DATA_DIR`.

### `ki show <source-id>`

Print the source summary, every feature, and counts for an indexed source.

### `ki query <term>`

Case-insensitive search across feature names/summaries and function-unit names/descriptions. Limit to one source with `--source <id>`.

```bash
ki query "rate limit"
ki query auth --source my-repo
```

### `ki export [source-id]`

Re-emit existing indices to markdown or YAML without re-running the pipeline. Omit `source-id` to export all.

```bash
ki export                              # all sources, markdown, into ./
ki export my-repo --format both -o docs
```

## MCP server

The package also ships an MCP server (`ki mcp`) exposing the same operations as tools (`index_source`, `list_sources`, `show_source`, `query`, `export_source`). The server uses **its own configured LLM** for summarization — the host's LLM (MCP sampling) is **not** invoked. This means an MCP host running Claude Code can drive indexing against a local Ollama model with no host-side cost.

## Data model

Each indexed source becomes a `KnowledgeIndex`:

```
Source
└── Feature[]              # logical grouping (e.g. a directory or module)
    └── FunctionUnit[]     # function | class | endpoint | section | ticket | other
```

Schemas live in `src/types/` and use Zod as the single source of truth — TypeScript types are derived via `z.infer`.

## Architecture

```
connectors/  →  indexer/  →  summarizer/  →  storage/  →  output/
                                                         │
                                       cli/  ────────────┘
```

- **connectors/** — pluggable adapters that scan a target and produce raw units. Currently only `git` (file-tree based) is implemented; `confluence` and `jira` are reserved in the type enum.
- **indexer/** — orchestrates scanning, diff detection against the previous index, and pipeline assembly.
- **summarizer/** — provider-agnostic LLM client (`LlmClient` interface) with implementations for Anthropic (`direct-client.ts`) and OpenAI-compatible endpoints (`openai-client.ts`); rate limiting via `p-limit` + tokens/minute; prompt templates.
- **storage/** — JSON persistence per source under `KI_DATA_DIR`, plus a YAML exporter.
- **output/** — renders the index to `KNOWLEDGE.md` via templates.
- **cli/** — `commander` entrypoints; every action is wrapped in `runAction` for uniform error handling.

## Development

```bash
npm run dev -- index ./some-repo   # run CLI from source
npm test                           # vitest, unit tests under tests/unit/
npm run test:coverage
npm run build                      # tsc → dist/
```

Tests are colocated by module under `tests/unit/<area>/<file>.test.ts`.
